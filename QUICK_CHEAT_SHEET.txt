7 ALGORITHMS - WHEN TO USE EACH
================================================================================

BFS (Breadth-First Search)
  "How does threat spread level by level?"
  When: Tracing infection ripples
  Speed: ⚡⚡⚡ VERY FAST
  
DFS (Depth-First Search)
  "How deep can an attack go?"
  When: Finding hidden paths
  Speed: ⚡⚡⚡ VERY FAST
  
DIJKSTRA (Shortest Path)
  "What's the best route?"
  When: Finding optimal paths
  Speed: ⚡⚡ Medium
  
GREEDY (Quick Decision)
  "Emergency! Which systems first?"
  When: Under active attack
  Speed: ⚡⚡⚡⚡ FASTEST
  
KNAPSACK (Perfect Solution)
  "Best tools for the budget?"
  When: Planning (take time for perfect answer)
  Speed: ⚡⚡ Medium
  
TSP (Route Planning)
  "Best order to visit all?"
  When: Audit path optimization
  Speed: ⚡⚡⚡ Fast
  
BRANCH & BOUND (Exact Optimization)
  "Absolute best solution?"
  When: Critical decisions
  Speed: ⚡⚡ Medium

PARAMETER QUICK REFERENCE
================================================================================

For BFS/DFS:
  → Start Node: Which computer? (e.g., node_0)

For DIJKSTRA:
  → Start Node: Where begin? (e.g., node_0)
  → End Node: Where go? (e.g., node_15)

For GREEDY/KNAPSACK/B&B:
  → Budget/Capacity: How much spend? (e.g., 3000)

For TSP:
  → Start Node: Which city start? (e.g., node_0)

Network Topology Options:
  → Random: Completely random connections (chaos)
  → Scale-Free: Few hub servers + many edge servers (realistic)
  → Small-World: Clustered communities (like departments)

================================================================================
METRICS EXPLAINED
================================================================================

CURRENT NODE    Which computer now?       Example: node_2
VISITED NODES   How many explored?        Example: 5
FRONTIER NODES  How many waiting?         Example: 8
COST            Total accumulated cost    Example: 45.5
TOTAL STEPS     Algorithm decisions made  Example: 20

DENSITY         Connections ratio         0.18 = 18% connected
AVG DEGREE      Average connections/node  3.5 connections each

STUDENT ACTIVITY IDEAS
================================================================================

Activity 1 - Prediction Game (20 min)
  "I'll run BFS. Which node visited in step 5?"
  Run it. Score points for correct predictions.

Activity 2 - Algorithm Detective (20 min)
  Show results without saying which algorithm.
  Clue 1: Took 8 steps
  Clue 2: Final value 80% optimal
  "Which algorithm?"

Activity 3 - Scenario Design (30 min)
  "Create a scenario where YOU would use each algorithm"
  Write it. Present to class.

Activity 4 - Speed Challenge (20 min)
  Team A: Run Greedy (goal: fastest)
  Team B: Run Knapsack (goal: best quality)
  Compare results. Discuss when each matters.
