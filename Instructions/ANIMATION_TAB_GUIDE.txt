================================================================================
HOW TO USE ANIMATION TAB - COMPLETE GUIDE WITH EXAMPLES
================================================================================

The Animation Tab can be confusing at first because it works differently than
you might expect. This guide explains EXACTLY how to use it with examples.

================================================================================
QUICK START: 3 WAYS TO USE ANIMATION TAB
================================================================================

METHOD 1: MANUAL STEPPING (MOST RELIABLE) ⭐
─────────────────────────────────────────────

What to do:
  1. Run algorithm in Control tab (don't click Play yet)
  2. Go to Animation tab
  3. Use the slider or First/Last buttons to navigate
  4. Read the metrics that appear for each step
  
How it looks:
  [⏮️ First]  [Slider: ●────────]  [⏭️ Last]
  
  Step 1 of 20
  Current: node_0 | Visited: 1 | Frontier: 5 | Cost: 0.0
  
  Move slider → Step 2 of 20
  Current: node_2 | Visited: 2 | Frontier: 7 | Cost: 2.5
  
  Click Last → Step 20 of 20
  Current: node_15 | Visited: 18 | Frontier: 0 | Cost: 87.5

Why this works best:
  ✓ You control the speed
  ✓ No timing issues
  ✓ Can inspect each step carefully
  ✓ Can pause and think
  ✓ Most reliable method

================================================================================

METHOD 2: AUTOMATIC PLAYBACK (WORKS BUT WITH LIMITATIONS) ⚠️
──────────────────────────────────────────────────────────

What to do:
  1. Run algorithm in Control tab
  2. Go to Animation tab
  3. Set "Animation Speed" slider:
     - 0.5 = Very slow (easiest to see)
     - 1.0 = Normal (recommended)
     - 2.0 = Fast
  4. Click "▶️ Play" button
  5. Watch what happens

What you'll see:
  - Dashboard will pause for a moment
  - Colors in Network tab might change slightly
  - Metrics will update
  - Then it will either:
    Option A: Finish animation in Network tab
    Option B: Show error and stop

Limitations:
  ⚠️ May not show smooth animation
  ⚠️ Colors might not change for each step
  ⚠️ Works best on small networks (10-15 nodes)
  ⚠️ May freeze briefly
  
When to use this:
  - Testing if it works
  - Small networks only
  - When you want fast overview

================================================================================

METHOD 3: MANUAL + ANIMATION TAB TOGETHER (BEST EXPERIENCE)
────────────────────────────────────────────────────────────

What to do:
  1. Run algorithm (doesn't matter if you click Play)
  2. Go to Animation tab
  3. Watch the slider and metrics
  4. Use slider to step through manually
  5. Watch Network tab simultaneously (optional but cool)
  
  OR
  
  1. Go to Network tab
  2. Watch network visualization
  3. Go to Animation tab
  4. Slide through steps
  5. See metrics for each step in Animation tab
  6. Understand the algorithm step-by-step

The BEST approach:
  ✓ Have Network tab and Animation tab visible (split view if possible)
  ✓ Use slider in Animation tab
  ✓ Watch colors change in Network tab
  ✓ Read metrics in Animation tab
  ✓ Complete understanding!

COMPLETE EXAMPLE: BFS ANIMATION STEP-BY-STEP
================================================================================

SETUP:
  Network: 10 nodes, random topology
  Algorithm: BFS
  Start Node: node_0

STEP-BY-STEP WALKTHROUGH:

Step 1 of 20:
  Current Node: node_0
  Visited Nodes: 1 (just node_0)
  Frontier Nodes: 5 (neighbors of node_0)
  Cost: 0.0
  
  → What this means: Starting point, exploring from node_0
  → Network shows: node_0 is YELLOW (current), others are GREEN

Step 2 of 20:
  Current Node: node_2
  Visited Nodes: 2 (node_0, node_2)
  Frontier Nodes: 7 (added node_2's neighbors)
  Cost: 1.0
  
  → What this means: Moved to node_2, checking its neighbors
  → Network shows: node_0 is RED (visited), node_2 is YELLOW

Step 3 of 20:
  Current Node: node_5
  Visited Nodes: 3 (node_0, node_2, node_5)
  Frontier Nodes: 8
  Cost: 2.0
  
  → What this means: Now at node_5
  → Network shows: node_0, node_2 RED (visited), node_5 YELLOW

...continue through all 20 steps...

Step 20 of 20 (FINAL):
  Current Node: node_15
  Visited Nodes: 10 (all reachable nodes)
  Frontier Nodes: 0 (done!)
  Cost: 45.5
  
  → What this means: Algorithm finished, explored all 10 nodes
  → Network shows: All nodes RED (visited), none YELLOW

WHAT YOU LEARNED:
  - BFS explored level-by-level
  - Each step visited one new node
  - Frontier grew as we found new neighbors
  - Cost accumulated as we moved
  - Total 20 steps to explore 10 nodes

================================================================================
COLOR MEANINGS WHILE ANIMATING
================================================================================

🟢 GREEN Node
  ├─ Status: Not visited yet
  ├─ Meaning: Algorithm hasn't looked at this node yet
  └─ In Animation tab: Shows in "Frontier" or not yet discovered

🟡 YELLOW Node
  ├─ Status: Currently being processed
  ├─ Meaning: Algorithm is looking at THIS node right now
  ├─ Location: Network tab shows this as bright yellow
  └─ In Animation tab: Shows in "Current Node" field

🔴 RED Node
  ├─ Status: Already visited
  ├─ Meaning: Algorithm processed this node and moved on
  └─ In Animation tab: Shows in "Visited Nodes" count

White/Gray Lines
  ├─ Connections between nodes
  └─ Show the network structure

================================================================================
STEP INFORMATION EXPLAINED
================================================================================

CURRENT NODE
  What: Which computer is algorithm processing RIGHT NOW?
  Example: "node_2"
  Why: Shows you WHERE the algorithm is
  How it changes: Moves to a different node each step
  
  In BFS: Moves systematically level by level
  In DFS: Moves deep then backtracks
  In Dijkstra: Moves to nearest unvisited

VISITED NODES
  What: How many nodes has algorithm already processed?
  Example: "5"
  Why: Shows PROGRESS through the network
  How it changes: Increases by 1 each step (usually)
  
  Graph: Creates line going up in Analytics tab
  Endpoint: Final count = total nodes explored

FRONTIER NODES
  What: How many nodes are waiting to be processed?
  Example: "8"
  Why: Shows how many OPTIONS algorithm has
  How it changes: Goes up when discovering neighbors, down when processing
  
  Large frontier: Many neighbors discovered
  Small frontier: Running out of options
  Zero frontier: DONE (no more nodes to process)

COST
  What: Total accumulated cost/distance?
  Example: "47.5"
  Why: Shows EFFICIENCY of the path
  How it changes: Increases as we move through edges
  
  For BFS/DFS: Usually low (no weights)
  For Dijkstra: Increases along optimal path
  For Greedy: Highest priority items selected

HOW TO PROPERLY USE THE ANIMATION TAB - PROCEDURE
================================================================================

PROCEDURE FOR MAXIMUM UNDERSTANDING:

1. CREATE NETWORK (Network Tab)
   └─ 15 nodes, "small-world" topology (good for seeing structure)
   └─ Click "✨ Create Network"
   └─ Wait for network graph to appear

2. RUN ALGORITHM (Control Tab)
   └─ Select: BFS (simplest to understand first)
   └─ Start Node: node_0
   └─ Click "Run Algorithm"
   └─ Wait for "✅ BFS completed"

3. OPEN ANIMATION TAB
   └─ You see: Step 1 of X
   └─ Current Node shows which node
   └─ Metrics show step info

4. STEP THROUGH MANUALLY (BEST METHOD)
   └─ Look at Step 1 of X
   └─ Read: Current, Visited, Frontier, Cost
   └─ Move slider slightly right
   └─ Look at Step 2 of X
   └─ Compare: What changed?
   └─ Continue until Step X of X

5. UNDERSTAND THE PATTERN
   └─ BFS: Visited count increases steadily
   └─ Frontier grows then shrinks
   └─ Cost stays low (no weights)
   └─ Read the story step-by-step

6. CHECK NETWORK TAB
   └─ Go back to Network tab
   └─ Notice colors show visited nodes
   └─ Connection shows network structure
   └─ Understand how algorithm explored it

7. TRY ANOTHER ALGORITHM
   └─ Go to Control tab
   └─ Pick DFS instead of BFS
   └─ Run it
   └─ Go back to Animation tab
   └─ Notice: Same nodes visited, DIFFERENT ORDER!
   └─ Understand: Different strategies = Different paths

COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE 1: "Animation Tab shows 'Run an algorithm first'"
SOLUTION: 
  You haven't run an algorithm yet
  Step 1: Go to Network tab → Create Network
  Step 2: Go to Control tab → Run Algorithm
  Step 3: Go back to Animation tab → Now it will show

ISSUE 2: "Play button doesn't seem to do anything"
SOLUTION:
  Play button works but may be subtle
  Try this instead:
    a) Use the slider to step through manually (more reliable)
    b) If you want animation, use slower speed (0.5) first
    c) Try on smaller network (10 nodes instead of 50)

ISSUE 3: "Colors don't change in network when I use slider"
SOLUTION:
  Check Network tab simultaneously:
    a) Open Network tab in another browser window
    b) Go back to Animation tab and use slider
    c) Network tab won't auto-refresh, but Network IS tracking steps
  OR:
    Go to Network tab to see visualization with current step colored

ISSUE 4: "Slider doesn't move smoothly"
SOLUTION:
  This is normal - Streamlit rerenders on each slider movement
  Try:
    a) Use First/Last buttons instead of slider
    b) Manually type step number if possible
    c) Just accept slight delays

ISSUE 5: "Animation freezes"
SOLUTION:
  This happens with large networks
  Try:
    a) Use smaller network (10-15 nodes)
    b) Use slider method instead of Play button
    c) Close other applications to free memory
