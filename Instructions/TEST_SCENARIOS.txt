SECTION 1: BFS (BREADTH-FIRST SEARCH) - "Level-by-Level Exploration"
================================================================================

TEST SCENARIO 1.1: Small Network Reachability
----------------------------------------------
DESCRIPTION: Determine how far a threat can spread from a source computer

SETUP:
  Network Size: 10 nodes
  Topology: random
  Algorithm: BFS
  Start Node: node_0

EXPECTED BEHAVIOR:
  - Animation shows spreading layer by layer
  - Step 1: Only node_0 visited
  - Step 2-3: Direct neighbors of node_0 visited (typically 2-3 nodes)
  - Step 4-7: Neighbors of neighbors visited
  - All steps: More nodes added to frontier at each level
  
METRICS TO CHECK:
  Total Steps: ~8-10 (depends on network connectivity)
  Nodes Visited: 8-10 (most but not all nodes typically reachable)
  Cost: Low (typically 0-5)
  Pattern: Linear growth then plateau when no more neighbors
  
WHAT THIS TEACHES:
  "This is how a virus spreads - reaching nearby computers first, then
   their neighbors, level by level. BFS guarantees finding all reachable nodes."

SETUP:
  Network Size: 12 nodes
  Topology: random
  Algorithm: DFS
  Start Node: node_0

EXPECTED BEHAVIOR:
  - Explores one path all the way to a dead end
  - Backtracks to explore alternative paths
  - Visited nodes show deep penetration before exploring breadth

  "DFS explores one attack path completely before trying others. Useful for
   finding if there's a cycle (loop) that could trap defenders."

-----------------------------------------------
DESCRIPTION: Find optimal route to reach critical server

SETUP:
  Network Size: 15 nodes
  Topology: random
  Algorithm: Dijkstra
  Start Node: node_0
  End Node: node_14

EXPECTED BEHAVIOR:
  - Explores nodes in order of distance from start
  - Not level-by-level like BFS (accounts for edge weights)
  - Always picks closest unvisited node
  - Finds guaranteed shortest path

  Total Steps: 10-15
  Nodes Visited: 12-15
  Final Cost: Should be minimum possible
  Cost should be optimal (compared to BFS which may not be)
  
WHAT THIS TEACHES:
  "Dijkstra guarantees finding the shortest path. Each step picks the
   nearest node, ensuring optimality. Cost matters, not just distance."

SECTION 10: ASSESSMENT QUESTIONS FOR STUDENTS
================================================================================

After running these tests, ask students:

1. BFS UNDERSTANDING:
   "Why does BFS explore level by level? Draw what layer 3 means."
   Answer: Each layer is +1 distance from source
   
2. DFS vs BFS:
   "Both visited 15 nodes in ~15 steps. Why is the order different?"
   Answer: BFS: breadth first, DFS: depth first
   
3. DIJKSTRA:
   "Why did Dijkstra take a longer path than BFS?"
   Answer: Edge weights - some edges are cheaper even if longer
   
4. GREEDY:
   "Greedy completed in 10 steps, Knapsack in 80. Why so different?"
   Answer: Greedy quick decision, Knapsack tries all combinations
   
5. KNAPSACK:
   "What does 'backtracking' mean at the end of Knapsack?"
   Answer: Finding which items are in the optimal solution
   
6. TSP:
   "Why isn't TSP visiting nodes in order 0, 1, 2, 3...?"
   Answer: Nearest neighbor - goes to closest unvisited, not next
   
7. BRANCH & BOUND:
   "How does B&B find the same solution as Knapsack faster sometimes?"
   Answer: Pruning eliminates bad branches early
   
8. COMPLEXITY:
   "Which algorithm completed fastest? Which took longest? Why?"
   Answer: Greedy fastest (O(n)), algorithms vary by approach
   
9. REAL-WORLD:
   "You're under attack - choose Greedy or Knapsack? Why?"
   Answer: Greedy - no time for perfect solution
   
10. OPTIMIZATION:
    "You have 3 days to choose security tools. Now what?"
    Answer: Knapsack or B&B - time allows optimal solution
