QUICK START GUIDE FOR TEACHERS & STUDENTS
Cybersecurity Threat Detection & Response System

START HERE: 3 STEPS TO GET GOING
================================

STEP 1: Launch App
  > Open Command Prompt/Terminal
  > Type: python run.py
  > Wait for: "You can now view your Streamlit app in your browser"
  > Opens: http://localhost:8502

STEP 2: Create Network (Network Tab)
  > Number of Nodes: 20 (good for learning)
  > Topology: small-world (most realistic)
  > Click: "✨ Create Network"
  > Wait for: Green graph with nodes and connections

STEP 3: Run Algorithm (Control Tab)
  > Select: BFS (simplest to understand)
  > Start Node: node_0 (it will auto-select)
  > Click: "Run Algorithm"
  > Done! Now go to Animation tab to watch

   └─ Pick when: Finding hidden paths

================================================================================
TEACHING SEQUENCE (Recommended Order)
================================================================================

LESSON 1: Basic Graph Traversal (1 hour)
  Topic: How to explore a network
  Algorithms: BFS, then DFS
  Demo:
    1. Create 10-node network
    2. Run BFS → show level-by-level spread
    3. Reset and run DFS → show different order
    4. Question: "Same nodes visited, but HOW was different?"

LESSON 2: Optimal Paths (1 hour)
  Topic: Finding the best route
  Algorithm: Dijkstra
  Demo:
    1. Create 15-node network
    2. Set Start: node_0, End: node_14
    3. Run Dijkstra → show path
    4. Question: "Why this path and not shorter hops?"

LESSON 3: Rapid Decisions (1 hour)
  Topic: Security under pressure
  Algorithm: Greedy
  Demo:
    1. Create 20-node network
    2. Budget: 3000
    3. Run Greedy → watch quick selection
    4. Compare to: "What if we had more time?"

LESSON 4: Perfect Solutions (1.5 hours)
  Topic: When you CAN find the best answer
  Algorithms: Knapsack, Branch & Bound
  Demo:
    1. Create 15-node network
    2. Run Knapsack → slow but perfect
    3. Run B&B → slightly different approach
    4. Question: "Both get 100% optimal. Which do you use?"

LESSON 5: Route Planning (30 min)
  Topic: Visiting many locations efficiently
  Algorithm: TSP
  Demo:
    1. Create 20-node network
    2. Run TSP → watch it build tour
    3. Question: "Is this the absolute best route?"
    4. Answer: "No, but it's good and FAST"

LESSON 6: Comparison & Trade-offs (1 hour)
  Topic: Choosing the right algorithm
  All Algorithms
  Demo:
    1. Same network, run all 7 algorithms
    2. Compare: Speed, Quality, Steps
    3. Create decision chart on board:
       "Choose Greedy if: _________"
       "Choose Knapsack if: _________"
       etc.

================================================================================
INTERACTIVE STUDENT ACTIVITIES
================================================================================

ACTIVITY 1: Prediction Game (20 minutes)
  Setup: Show network without running algorithm
  Task 1: "I'll run BFS. Predict: which node visited in step 5?"
  Run: BFS
  Score: +1 for correct prediction
  
  Task 2: "Now DFS. Will it visit more or fewer nodes in first 5 steps?"
  Run: DFS
  Score: +1 for correct prediction
  
  Task 3: "For 20-node network, Greedy will take ___ steps"
  Run: Greedy
  Score: +1 if within 2 steps
  
  Winner: Most correct predictions

ACTIVITY 2: Algorithm Detective (30 minutes)
  Show: Results of an algorithm (without saying which one)
  Clue 1: "Took 8 steps"
  Clue 2: "Final value was ~80% optimal"
  Clue 3: "Budget was 2500"
  Question: "Which algorithm?"
  
  Repeat with different clues for all 7 algorithms

ACTIVITY 3: Scenario Design (40 minutes)
  Groups: Pair students
  Task: "Design a scenario where YOU would use each algorithm"
  Examples:
    - "BFS: Trace virus spread from one computer"
    - "Dijkstra: Route backup data through fastest network path"
    - "Greedy: Choose which systems to patch under DDoS"
  Requirement: Write scenario, predict outcomes
  Share: Present to class

ACTIVITY 4: Speed & Accuracy Challenge (30 minutes)
  Setup: 20-node network created on screen
  Team 1: Run Greedy (goal: fastest with reasonable solution)
  Team 2: Run Knapsack (goal: perfect solution)
  
  Measure:
    - Time to complete
    - Solution quality
    - Nodes protected
    - Cost efficiency
  
  Debrief: "When is each approach valuable?"

ACTIVITY 5: Real-World Case Study (50 minutes)
  Scenario: "Company network infected. Follow response."
  
  Phase 1 (10 min): Use BFS to map infection spread
  "How many computers can the virus reach from patient zero?"
  
  Phase 2 (10 min): Use Dijkstra to route response
  "What's the fastest path to isolate critical server?"
  
  Phase 3 (15 min): Use Greedy for emergency response
  "With $5000, which systems to patch first?"
  
  Phase 4 (10 min): Plan better with Knapsack
  "With the same $5000 and careful planning, better selection?"
  
  Debrief: "Which approach should they have used? Why?"

================================================================================
TROUBLESHOOTING - QUICK ANSWERS
================================================================================

Q: "Error on the Animation tab"
A: First create network (Network tab), then run algorithm (Control tab)

Q: "All tabs show error"
A: Go to Network tab and create a network first

Q: "Algorithm won't run"
A: Did you create network? (Network tab → Create Network)

Q: "Can't see colors changing in animation"
A: Click the "▶️ Play" button in Animation tab

Q: "Results look different each time"
A: Network is random. Create same topology/size for consistent results

Q: "Animation is too fast/too slow"
A: Use "Animation Speed" slider (0.5=slow, 2.0=fast)

Q: "My network is too small/large"
A: Change "Number of Nodes" slider. Try 10-30 for clarity.

Q: "Can I see the actual path Dijkstra found?"
A: Use Animation tab - colors show visited nodes. Read the path.

SAMPLE 1-HOUR LESSON PLAN
================================================================================

TITLE: "Finding Hidden Threats in Computer Networks"
GRADE: High School or College
TIME: 60 minutes

INTRO (5 min):
  - Show network diagram on screen
  - "This is a company's computer network"
  - "Attackers can spread through it. How do we trace them?"
  - "Graph algorithms help us find threats fast"

DEMO 1 (10 min): BFS - "How does infection spread?"
  - Show 15-node network
  - Run BFS from central computer
  - Show wave-like spreading
  - "Level by level, like ripples in water"

DEMO 2 (10 min): Dijkstra - "What's the fastest path?"
  - Show same network
  - Find path from server A to server B
  - Highlight the optimal route
  - "Not always the most direct - depends on network congestion"

DEMO 3 (10 min): Greedy - "Emergency patch selection"
  - Show budget of $3000
  - Algorithm selects most critical systems
  - "When you're under attack, you don't have time to be perfect"

COMPARISON (10 min):
  - Show same problem with Greedy vs Knapsack
  - Greedy: Fast, ~80% optimal
  - Knapsack: Slower, 100% optimal
  - Discussion: "When do we accept 80%? When do we need 100%?"

ACTIVITY (10 min): Student prediction
  - Show network
  - Ask: "If we run BFS on this, how many steps?"
  - Run it
  - Award points for close predictions

CLOSING (5 min):
  - "Algorithms are tools. Different tools for different jobs."
  - "Security professionals must know which tool to use."
  - "This system shows when and how to use each."

GETTING STUDENTS EXCITED - ENGAGEMENT TIPS
================================================================================

TIP 1: Use Real Stories
  Instead of: "Run BFS"
  Say: "We just found a virus. Use BFS to see how far it spreads."

TIP 2: Make It Competitive
  "Team A uses Greedy, Team B uses Knapsack. Who finds better solution?"

TIP 3: Ask Predictions First
  "Guess: Will DFS or BFS finish first? Why?" (Then show)

TIP 4: Connect to News
  "This is how hackers do reconnaissance. This is how defenders respond."

TIP 5: Let Them Design Scenarios
  "Create a network. Choose an algorithm. Why did you pick it?"

TIP 6: Show Impact
  "These algorithms protect real companies from real attacks."

TIP 7: Make It Interactive
  Let students try running algorithms themselves
  Let them create networks of different sizes
  Let them predict outcomes before running
