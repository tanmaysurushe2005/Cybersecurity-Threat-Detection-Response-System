QUICK PARAMETER GUIDE
================================================================================

BFS/DFS PARAMETERS:
  Start Node: Which computer to begin from?
             Example: node_0 (main server)

DIJKSTRA PARAMETERS:
  Start Node: Where to begin?
  End Node:   Where to reach?
  Example: Find path from node_0 to node_15

GREEDY/KNAPSACK/BRANCH&BOUND PARAMETERS:
  Budget/Capacity: How much to spend?
  Example: 3000 (dollars for security patches)

TSP PARAMETERS:
  Start Node: Which city to start tour from?
  Example: node_0
  (Automatically visits all others and returns)

================================================================================
COLOR MEANINGS IN ANIMATION
================================================================================

🟢 GREEN
   Meaning: Normal node, not visited yet
   Action: Will be visited next

🟡 YELLOW
   Meaning: Currently being processed
   Action: Algorithm is looking at this node right now
   Duration: 1 step

🔴 RED
   Meaning: Already visited
   Action: Won't be visited again (algorithm done with it)

WHAT THE COLORS TELL YOU:
  Green nodes = "Still to explore"
  Yellow node = "Looking at this now"
  Red nodes  = "Already explored"

================================================================================
METRICS EXPLAINED
================================================================================

CURRENT NODE
  What: Which computer is algorithm processing right now?
  Example: "node_2"
  Why: Shows where algorithm is in execution

VISITED NODES
  What: How many computers have been explored?
  Example: "5"
  Why: Shows progress toward exploring whole network

FRONTIER NODES
  What: How many computers are waiting to be explored?
  Example: "8"
  Why: Shows how many options algorithm has

COST
  What: Total accumulated cost/distance so far
  Example: "45.5"
  Why: Shows efficiency of path taken

TOTAL STEPS
  What: How many decisions did algorithm make?
  Example: "20"
  Why: Indicator of algorithm complexity

================================================================================
KEY CONCEPTS TO REMEMBER
================================================================================

1. GRAPH
   What: Network of connected nodes (computers)
   Visual: Dots (nodes) with lines (edges/connections)
   
2. NODE
   What: One computer in the network
   Visual: Green dot (unvisited), yellow (current), red (visited)

3. EDGE
   What: Connection between two computers
   Visual: Line connecting two dots

4. ALGORITHM
   What: Step-by-step procedure to solve a problem
   Example: "Visit all nodes level by level" (BFS)

5. OPTIMAL
   What: Best possible solution
   Example: "Shortest path is optimal"

6. HEURISTIC
   What: "Good enough" solution that's fast
   Example: "Pick closest node next" (nearest neighbor)

7. COMPLEXITY
   What: How time/space grows with input size
   Example: O(V+E) means doubles when network size doubles
